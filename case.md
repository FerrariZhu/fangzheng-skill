## 关键决策故事（访谈）

### 故事 A — 途虎超级会员：先核心链路，体验后补

**对应问题**：时间紧、领导/CEO 强推、体验与进度怎么取舍？

**局面**
- CEO 亲点要做超级会员；约 8 月中立项，需在国庆前上线，赶黄金周
- 时间紧、任务重：评估需跨约 **16 个团队**协作，总成本约 **200+ PD**

**判断与选择**
- 为赶上进度，**优先跑通核心链路**：支付、下单、权益发放等
- 用户体验暂时让步：例如购后短时间无法自助退费；用户来电退卡需客服手动进系统填工单，链路长、响应慢

**结果**
- 主流程按时跑通，赶上国庆节奏
- 退费等体验在主流程跑通后**后续快速迭代补上**

**启示**
- 业务高压下默认「先保短期指标/进度，体验后补」；但后续必须排期还债，不能永久裸奔
- 职业素养上仍认同体验应第一——参谋时诚实呈现这对张力

**对应框架**：冲突取舍（体验 vs 指标）；#1 目标转化；#4 主流程优先

### 故事 B — SHEIN：AB 验证后再组件化平台

**对应问题**：营销玩法要不要直接做成配置化中台？

**局面**
- 算法定价已完成：用户可按身份/地区获得不同价格
- 场域内缺营销氛围感；对标拼多多、特步等竞对，营销氛围更强
- 不确定动效弹窗是否真有效；竞对大约每周一版新迭代，节奏快

**判断与选择**
1. **先 MVP 验证**：把静态弹窗改成约 **60 秒**动效弹窗；验证 **2～3** 个弹窗版本
2. 每次 A/B 结果均为**正向**
3. **再平台化**：为追竞对迭代速度，搭建组件库——弹窗拆成多个片段，配置后台组合后前端投放

**结果**
- 用实验证明氛围/动效有效，再投入组件化能力建设，避免一上来就大中台赌方向

**启示**
- 先 MVP + AB，再沉淀可复用（反对「一上来就大中台」）
- 组件化目的是**运营节奏与竞对速度**，不是技术优雅本身

**对应框架**：不变原则（先 MVP 再可复用）；#2 人群/策略；#3 工具化；冲突取舍（MVP vs 平台）

## 1. CTW：平台 Admin MCP 化

**公司/产品**：CTW 株式会社（g123.jp），游戏宣发配置平台  
**时间**：2026.4～

### 背景

Admin 是全平台游戏投放配置后台。公司在建内部 Agent D 系统，目标是用自然语言完成配置动作，替代传统表单式后台。

### 动作

1. 调研系统基础能力与市场同学真实场景
2. 抽象 **4 大核心流程 + 8 个分支流程**；模拟真实对话，整理核心 query
3. 整理出约 **37 个 tools**，覆盖增删改查与排序等能力
4. 持续优化 tool 描述，提高自然语言下的命中率

### 结果

- 约 1 个多月整理、开发、优化后上线
- 后台能力拓展不再强依赖后台前端开发步骤
- 配置时长约 **30 分钟 → 15 分钟**；学习成本明显下降

### 可复用决策启示

- Agent 化第一步不是「接 LLM」，而是**流程与场景抽象**
- Tool 粒度要覆盖真实操作闭环；描述质量决定命中率
- 成功指标用**时长 + 学习成本 + 是否去掉前端开发瓶颈**，不只是「能聊天」

**对应框架**：#6 Agent 化；#7 AI 全链路交付

## 2. 携程 Trip：美食林 SEO 增长

**公司/产品**：携程集团，Trip.com 美食林  
**时间**：2025.5～2026.4

### 背景

入职后先偏 C 端优化，后业务调整侧重 T 端 SEO。起点流量偏低，需要可复利的搜索增长路径。

### 动作

1. 研究谷歌规则与竞品；优化页面 TDK（标题/描述结构）、餐厅结构化数据字段、URL 结构
2. 做 SEO 基建：将已优化内容生成静态链接，并在核心站点投放 sitemap

### 结果

- SEO 流量（月日均）约从年初 **215 → 12,919**（截止约 11 月）
- TDK 优化带来 SEO 场景 UV 环比约 **+5,100**
- 静态链接 + sitemap 投放带来 SEO 场景 UV 环比约 **+5,000**

### 可复用决策启示

- 增长前先补**可复利基建**（结构化数据、URL、sitemap、静态资产）
- 把优化拆成可归因动作，方便看哪一杠杆在起作用
- 规则与竞品研究是动作设计的输入，不是单独的「调研产出」

**对应框架**：#5 基建先于增长杠杆

## 3. SHEIN CLUB：付费会员频道页转化

**公司/产品**：SHEIN，CLUB 付费会员频道页（owner）  
**时间**：2023.10～2025.5

### 背景

频道页是付费会员产品包的核心转化场景，需同时承接站内流量转化与权益介绍。要从**功能 + 运营策略**两侧做场域内目标转化。

### 动作

1. **算法能力**：人群精细化定价；商品流接入算法推荐，抬开卡转化
2. **弹窗动效**：按人群画像拆解利诱点；目标/非目标人群分叉；动效弹窗替代静态弹窗，做到「专属弹窗」
3. **组件化**：竞对营销氛围迭代快 → 抽象现有内容为可配置组件，快速跟进权益与投放

### 结果

- 24 年购买页开卡转化率约提升 **100%（0.8%→1.69%）**
- 算法定价迭代至约 8.0、推荐至约 3.0；US 站开卡转化累计约 **+40%**，并拓展非 US
- 目标人群弹窗约 4.0、非目标约 2.0；US 站开卡转化累计约 **+30%**
- 频道页改版优首屏；US 站约 **+10%**
- 页面全球 UV 约 50 万；在籍会员约 200 万

### 可复用决策启示

- 先钉死场域 KPI（开卡转化），再并行打算法、触达、页面效率
- **目标/非目标分叉**比统一体验更能抬转化
- 组件化是为了**运营节奏**，不是为了技术优雅
- 算法定价 + 推荐是转化杠杆，需与页面结构一起迭代
- 过程细节见上方**故事 B**（先 AB 再组件化）

**对应框架**：#1 目标转化优先；#2 人群拆分；#3 工具化/配置化（组件）

## 4. 途虎：付费会员体系

**公司/产品**：途虎养车，付费会员（黑卡 → 超级会员等）  
**时间**：2021.06～2023.10

### 背景

综合权益会员卡：用户用权益与优惠延长 LTV，平台用会员锁客。需覆盖多品类诉求，并发挥门店网络分销能力。

### 动作

1. 迭代途虎黑卡，拓展多种卡模板（洗车版、保养版、门店版等）
2. 跟进竞对：次卡形态迭代为以权益折扣为核心的超级会员；线下试点正向后推向全国
3. 重设线上线下分销规则，提高门店销售积极性

### 结果

- 多种付费会员卡版本覆盖平台核心商品
- 超级会员试点正向，计划全国推广（简历表述）
- 在职期间全站付费会员约 **80 万**量级；另一处表述在籍约 **90 万**左右

### 可复用决策启示

- 会员产品要**模板化**，用多卡型覆盖不同诉求，而不是一张万能卡硬推
- O2O 场景下，**分销规则与门店激励**和权益设计同等重要
- 重大形态切换用试点城市验证，再全国铺开
- 高压排期细节见上方**故事 A**（超级会员：先核心链路，体验后补）

**对应框架**：#1 目标转化 / 锁客 LTV；#4 在现有门店网络上增强分销

## 5. 途虎：营销工具 — 组合售卖

**公司/产品**：途虎养车，跨业务组合售卖 / 券包工具  
**时间**：2021.06～2023.10

### 背景

各业务线烟囱式发展，跨业务打包售卖无法支持。需要可复用工具，打通组合套餐能力。

### 动作

1. 抽象复用性强的工具：接入活动页、优惠券等基础营销能力；券码卡类虚拟商品的创建与履约收口；前端提供使用能力，降低业务线接入成本
2. 枚举并定义 **五类商品形态**，支持任意组合打包
3. 策略：不改变现有业务流程；打包时做能力增强（随单使用、权益续费等）；拓展场景（搭售、外投、批量采买）；优化使用体验（引导与承接页）
4. 拉通结算、订单、履约、财务、BI，降低后续接入成本

### 结果

- 洗美主链路接入券包随单售卖，业务侧开发成本约仅 **7pd**；抖音等三方售卖底层复用
- 接入业务场景约 **21 个**，承接大型活动约 **3 次**
- 22 年帮助业务线销售额约 **598.7 万**，发券约 1600 万张，后转约 **5000 万+**

### 可复用决策启示

- 平台型问题优先**抽象商品与履约模型**，而不是为单次活动定制
- 「不改主流程 + 能力增强」是多业务线协同时的默认策略
- 成功看**接入成本、场景数、后转**，不只看工具本身上线

**对应框架**：#3 工具化与平台复用；#4 不改主流程的能力增强
